# Getting Started

This guide walks you through signing in for the first time and installing the tools you need to work with OpenDataDSL.

## Onboarding

### Sign in to the portal

Go to the portal and sign in using your corporate Microsoft credentials (Azure Active Directory):

<a className="btn btn-primary" href="https://portal.opendatadsl.com">Portal — Click Sign In</a>

After signing in, you will be asked to accept the application terms.

### Company details

On first sign-in, you will be prompted for a few details about your company. This only needs to be done once and can be edited later in the portal.

### Personal details

You will also be asked for some basic information about yourself. Any other users from your company who sign in will be prompted for the same. This can also be edited later.

## User Interfaces

OpenDataDSL provides three free interfaces for interacting with the platform. You can use any combination depending on your workflow.

### Web Portal

The web portal is a browser-based interface to the OpenDataDSL platform. It works on desktop and mobile, and lets you manage your account, browse data, and run processes.

<a className="btn btn-primary" href="/docs/user/portal">Instructions for using the Web Portal</a>

### Excel Add-in

The Excel add-in connects your spreadsheets to the OpenDataDSL platform, letting you search for data, pull it into Excel, and write data back to the platform.

<a className="btn btn-primary" href="/docs/user/excel">Instructions for installing and using the Excel Add-in</a>

### VS Code Extension

The VS Code extension lets you write, debug, and execute ODSL code. It also lets you upload scripts and report formats to the platform for use in processes, reports, and Smart Curves.

<a className="btn btn-primary" href="/docs/user/vscode">Instructions for installing and using the VS Code extension</a>
